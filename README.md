# Notification Service

## Overview

The **Notification Service** is an event-driven microservice responsible for processing asynchronous notifications across the E-Learn Microservices Platform. It consumes events from Apache Kafka and delivers notifications such as welcome emails, purchase confirmations, password reset emails, and future system notifications.

The service is designed to operate independently of business services, ensuring that notification processing does not impact API response times.

## Role in E-Learn Platform

The Notification Service enables asynchronous communication across the E-Learn platform by consuming Kafka events and delivering notifications independently of business services. This event-driven architecture improves system scalability, responsiveness, and maintainability by decoupling notification processing from core business logic.
